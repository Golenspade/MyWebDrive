# 低成本高画面“假药”制作技术栈（摇轮椅教程）

适用人群：希望以较低成本完成“高画面感”的叙事视频/演出效果，基于 WebGAL 生态与常见创作工具的创作者。

> 注：文中对“假药”等词汇均为社区内梗称呼，与医疗无关。

## 硬件需求

- 一台 Windows 电脑（建议参考 5k+ 人民币的主流游戏本配置；台式机预算可更低）
  - 能流畅运行：WebGAL、本地剪辑软件（如剪映）、特效合成（AE）或实时特效渲染（Shoost）
- Q1：WebGAL 也支持 macOS，为什么不用 Mac？
  - A1：包括 MyGO 2.3/2.4 引擎在内的许多二创流通素材仅有 Windows 版本；工作流中亦依赖多款 Windows 工具，Mac 上可能不可用。
- 一颗“充满灵感菇”的大脑（写不出来，一切免谈）。
  - P：也可与作者（如 蓝P、LOFTER 等平台）合作，在取得授权后进行视频化改编。

## 软件清单

- 写作工具：任意（手机备忘录即可；是否排版取决于是否同步发文）—作者示例使用 VS Code
- 线上大模型：将自然语言转为 YAML（AnoGO 可识别）
- AnoGO：将 YAML 转换为 WebGAL 可用脚本（参考 B 站 @明见）
- WebGAL 本体：可视化演出（参考 B 站 @邻家码农真昼酱）
- WebGAL MyGO Avemujica 2.3/2.4 专版：同上
- Live2D Watcher：快速检索表情动作与动画（参考 B 站 @猫猫猫宫）
- Photoshop：封面与模型制作（自学/教程众多）
- OBS Studio：录制底板视频与 Shoost 画面（教程众多）
- Shoost：实时渲染特效，提升氛围（付费；参考 B 站 @uv唐大头）
- AE：特效制作与演出增强（学习成本高、配置要求高）
- 剪映：最终合成 BGM/字幕/转场 等

> 链接请按工具名自行检索；B 站关键字见上方作者 ID。

## 工作流 Overview

1) 前期写作 → 2) 大模型结构化（YAML）→ 3) 用 AnoGO 生成 WebGAL 脚本 → 4) 导入 WebGAL 基础演出 → 5) Live2D Watcher 辅助查表情/动作并微调 → 6) Shoost 实时特效渲染 → 7) OBS 采集 Shoost → 8) 剪映后期合成 → 9) PS 封面 → 10) 发布

## Step 1：前期写作（强烈建议“类 YAML”）

- 若“视频+发文”双轨：直接自然语言写作即可；
- 若只做视频：强烈建议“类 YAML”格式（每段有标题，标题与内容用冒号分隔）。

### Q1：为什么用“类 YAML”？
- 大模型输出 YAML 的准确性更高，减少大模型擅自改写结构的倾向；
- 自查与比对更高效，定位错误更容易。

### Q2：如何写成“类 YAML”？
- 示例（伪 YAML，示意结构与语气；请按你文本实际改写）：

```text
羽丘走廊：
旁白：大车啊能不能放过我这一次...
角色：爱音
对话：别追啦，为什么追我？
角色：灯
对话：因为你有...不是，因为我需要 anon
```

- 严格 YAML 结构（建议作为目标输出样式）：

```yaml
- 背景: 地点名称
- 旁白: 旁白内容
- 角色: 人物名称
  动作: 情绪动作（生气/告别/哭泣/感动/决心/悲伤/认真/害羞/微笑/惊讶/思考）
  对话: 人物对话
```

### Q3：相对自然语言直接写的优势？
1. 更利于大模型结构化，降低误判；
2. 后续自查与修正更直观（逐条对比）。

## Step 2：用大模型将文章转为 YAML

为大模型准备清晰 Prompt，声明输入类型与期望输出结构。以下为可用 Prompt 模板（可按项目自定义）：

```md
# 结构化脚本生成器
- 目标：将输入小说文本分析并转换为 YAML。
- 技能：
  1) 精确中文处理（修正错别字/标点）；
  2) 上下文理解（地点、说话者、情感/动作推断；区分旁白/对话/心理活动）；
  3) 熟悉 YAML 输出格式。
- 要求：动作仅能从列表中选择（生气/告别/哭泣/感动/决心/悲伤/认真/害羞/微笑/惊讶/思考）；心理活动使用（）括起。
- 输出格式（YAML）：
  - 背景: 地点名称
  - 旁白: 旁白内容
  - 角色: 人物名称\n    动作: 情绪动作\n    对话: 人物对话
```

> 小贴士：将“写作大纲/剧情摘要/主题风格/人设/人称”等一并喂给模型可显著提升一致性。不同项目建议分开对话，避免上下文串台。

- Q：选哪家的大模型？
  - A：此任务需要强上下文推断与较大 token 上下文。优先：Grok / Gemini（需科学）；其次：Claude / ChatGPT（需科学）；DeepSeek 亦可，但上下文较弱，长文建议分段处理。

## Step 3：将 YAML 喂给 AnoGO，生成 WebGAL 脚本

- 参考 B 站 @明见 教程；
- 操作：将 Step 2 的 YAML 粘贴到 AnoGO，导出 WebGAL 可识别脚本。

## Step 4：导入 WebGAL 并进行可视化微调

- 将 AnoGO 输出脚本粘贴到 WebGAL 项目；
- 同时打开原文与 Live2D Watcher：
  - 快速检索表情/动作/角色动画；
  - 对台词编排、立绘与背景的变换进行微调；
  - 舞台效果设置与过渡。

> Q：为何使用 Live2D Watcher？
> A：WebGAL 实时预览对立绘变换/动作的反馈存在一定问题；Watcher 的检索与预览更稳定高效。

## Step 5：使用 Shoost 渲染实时特效

- 参考 B 站 @uv唐大头 的相关教程；
- 步骤：导入 WebGAL 画面 → 设置灯光/运镜/特效 → 预览 → 定稿；
- Q：为什么用 Shoost？
  - A：相比 AE 学习成本更低，能快速完成较为“高级”的氛围特效；
- Q：Shoost 的限制？
  - A：
    1) 特效参数在演出中不可动态改动，背景切换需重渲；
    2) 作为实时渲染，对性能有要求（分辨率调低可缓解掉帧）；
    3) 字幕能力有限，复杂字幕请在后期软件中完成。

## Step 6：用 OBS Studio 捕获 Shoost 画面

- OBS 为主流录屏工具，稳定、强大、上手快；
- 采集 Shoost 输出，设置码率/分辨率/帧率以匹配目标平台要求。

## Step 7：剪映进行后期合成

- 合成 BGM/字幕/转场/滤镜/节奏；
- 建议建立片头/片尾/转场的可复用模版，提高效率与一致性。

## Step 8：封面制作（Photoshop）

- 如果要更“有质感”的封面：
  1) 在 WebGAL 中导入本期主要立绘；
  2) 在 Shoost 中渲染你需要的特效；
  3) 截图导出；
  4) 在 PS 中进行合成与排版。
- 相比“WebGAL+PS”直出，Shoost 特效通常更具氛围感。

## Step 9：发布

- 以 B 站为例：上传成片与封面，完善分区/标签/简介；
- 复盘与复用：沉淀本期的 YAML/脚本/项目预设，成为下期起点。

## FAQ（补充）

- Q：大模型输出是否绝对准确？
  - A：不完全。尤其“指向性不强”的句子（如“她什么都没说”）会被模型自行理解。务必在导出 YAML 后逐条检查。
- Q：可以改动 Prompt 吗？
  - A：当然。Prompt 的作用是让模型更好理解你的“目标与约束”。建议按项目做专属 Prompt，并提前提供大纲/摘要/风格/人设。
