# 常见问题 (FAQ)

这里汇总了用户最常遇到的问题和解决方案。

## 🚀 入门问题

### 如何开始使用平台？

1. 访问 [快速入门指南](/docs/getting-started)
2. 下载所需的引擎和工具
3. 阅读相关教程
4. 开始创作你的第一个项目

### 需要什么技术背景？

- **基础使用**: 不需要编程基础
- **进阶功能**: 建议了解基本的脚本语法
- **高级开发**: 需要 JavaScript/TypeScript 知识

### 平台是免费的吗？

- **引擎和工具**: 完全免费
- **基础素材**: 免费提供
- **高级素材**: 部分付费
- **商业使用**: 遵守相应授权协议

## 🎮 引擎相关

### WebGAL 各版本有什么区别？

| 版本 | 特点 | 适用场景 |
|------|------|---------|
| 标准版 | 功能全面 | 通用项目 |
| MyGO 专版 | MyGO 主题优化 | MyGO 主题项目 |
| 东山版 | 高级功能 | 专业开发 |

详见 [版本对比](/docs/resources/version-control)

### 如何选择合适的版本？

- **新手**: 推荐标准版
- **MyGO 粉丝**: 推荐 MyGO 专版
- **专业开发**: 推荐东山版
- **不确定**: 先试用标准版

### 引擎支持哪些平台？

- ✅ Windows 10/11
- ✅ macOS 10.15+
- ✅ Linux (主流发行版)
- ✅ Web 浏览器
- ✅ 移动端 (通过打包)

## 📥 下载问题

### 下载速度很慢怎么办？

1. 更换下载时间段
2. 使用下载工具（IDM、aria2）
3. 尝试镜像站点
4. 检查网络连接

详见 [下载指南](/docs/resources/download)

### 下载的文件打不开？

**可能原因**:
- 文件未完全下载
- 解压工具不支持格式
- 文件损坏

**解决方案**:
1. 重新下载文件
2. 使用 7-Zip 或 WinRAR
3. 验证文件哈希值

### 如何验证文件完整性？

```bash
# Windows (PowerShell)
Get-FileHash -Algorithm SHA256 filename.zip

# macOS/Linux
shasum -a 256 filename.zip
```

对比官方提供的哈希值。

## 🛠️ 工具使用

### Live2D Watcher 无法加载模型？

**检查清单**:
- ✅ 模型文件完整
- ✅ 文件路径正确
- ✅ 格式支持 (.model3.json)
- ✅ 显卡驱动最新

详见 [Live2D Watcher 指南](/docs/guide/live2d-watcher)

### AnoGO 优化后图片质量下降？

**调整方法**:
1. 提高压缩质量参数
2. 使用无损压缩模式
3. 调整输出格式设置

```bash
# 高质量压缩
anogo optimize --quality=95 --format=png
```

### Bandori Craft 导出失败？

**常见原因**:
- 磁盘空间不足
- 导出路径权限问题
- 资源文件过大

**解决方案**:
1. 清理磁盘空间
2. 更换导出路径
3. 降低导出质量

## ⚡ 性能问题

### 游戏运行卡顿？

**优化建议**:
1. 压缩图片资源
2. 降低音频码率
3. 减少同时加载的资源
4. 启用资源预加载

详见 [性能优化](/docs/best-practices/performance)

### 加载时间过长？

**改进方法**:
1. 使用 WebP 格式图片
2. 启用资源压缩
3. 实现延迟加载
4. 优化资源大小

### 内存占用过高？

**解决方案**:
1. 及时释放不用的资源
2. 优化图片尺寸
3. 减少缓存数量
4. 使用对象池

## 🔧 开发问题

### 脚本语法错误？

**常见错误**:
```javascript
// ❌ 错误: 缺少分号
changeBg:background.jpg

// ✅ 正确
changeBg:background.jpg;

// ❌ 错误: 路径错误
changeFigure:/figure/character.png;

// ✅ 正确
changeFigure:character.png;
```

### 如何调试项目？

1. 使用浏览器开发者工具
2. 查看控制台错误信息
3. 使用东山工具集调试器
4. 添加日志输出

### 版本升级后项目无法运行？

**迁移步骤**:
1. 备份原项目
2. 查看更新日志
3. 按照迁移指南操作
4. 测试所有功能

详见 [版本管理](/docs/resources/version-control)
